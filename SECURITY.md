# Security Policy

## 管理后台

部署到公网前必须设置强随机 `ADMIN_TOKEN`，并只通过 HTTPS 访问 `/admin`。不要把 `.env`、`server-data/`、管理员令牌或学生提交数据提交到 Git。

## 学生提交数据

学生反馈可能包含联系方式、菜单照片和备注。部署者需要自行负责告知用户数据用途、定期备份、删除无用附件，并限制后台访问权限。

## 数据来源

项目内置食堂数据来自公开的西电后勤公众号正文菜单整理。综合楼、家属区和外卖商家应通过学生提交、商家授权或校内公开资料审核后加入，不建议绕过平台访问控制抓取第三方外卖页面。

## 漏洞反馈

如果发现安全问题，请在 GitHub 仓库新建 issue，并避免公开真实管理员令牌、学生联系方式或未脱敏数据。
